No stupid shit. If you're writing some code that would be flagged in a code review as brittle, or confusing, or hacky, then stop. Think of either:
- A simpler way to achieve the same goal that fits into our design constraints
- Is what you are doing necessary? If you were to work back from first principles, would you still be doing it? Or did you make some assumption earlier that forces this thing, but is not correct?

If you can't work with either, but what you're doing is still hacky and bad, then just stop. I prefer half finished code to hacky bullshit that I have to throw away. I GREATLY prefer that, actually. Stop, tell me the problem, and we'll work through it.

Otherwise, be diligent, and write simple, straightforward code. Work back from first principles when stuck. You are talented!
